# SKILL 优化总结 / Skill Optimization Summary

## 优化日期 / Optimization Date
2026-06-12

## 核心定位变更 / Core Positioning Change

### 之前 / Before
- **名称**: 世界杯预测机 / World Cup Predictor
- **定位**: 预测工具
- **重点**: 提供概率输出和预测结果

### 之后 / After
- **名称**: 世界杯预测市场研究助手 / World Cup Prediction Market Research Assistant
- **定位**: 严谨的概率分析师与风险管理顾问 / Rigorous probability analyst and risk management advisor
- **重点**: 结构化研究、风险分析、教育性决策框架

## 主要改进 / Major Improvements

### 1. 结构化输出模板 (8-Section Report)

新增了完整的 8 章节研究报告框架：

1. **数据摘要** - 数据版本、时间戳、声明
2. **隐含概率计算** - 去水方法、市场 vs 模型概率
3. **主观概率判断与依据** - 多因素综合、融合逻辑
4. **价值差异分析** - EV 计算、市场分歧标记
5. **风险因素** - 系统性风险、比赛特定风险、组合风险
6. **情景分析** - 最佳/最坏情况、预期收益、破产风险
7. **资金分配示例** - 保守/中性/激进三种方案（教育性）
8. **结论与研究性建议** - 核心发现、需进一步研究的问题

### 2. 工作流程优化 (5-Phase Workflow)

从简单的线性步骤升级为结构化的五阶段流程：

- **Phase 1**: 数据获取与校验
- **Phase 2**: 模型独立性保障 (blind-commit)
- **Phase 3**: 市场数据采集 (带用户确认)
- **Phase 4**: 概率计算与分析
- **Phase 5**: 结构化输出 (渐进式披露)

### 3. 行为准则体系 (Non-Negotiable Rules)

新增四大类严格规则：

#### 技术准确性 / Technical Accuracy
- 保留了原有的技术规则
- 强化了数据口径区分 (90min vs advance)
- 明确了审计追踪要求

#### 研究定位与合规 / Research Positioning & Compliance
- **7 个必须做**：明确分析性质、披露不确定性、展示计算过程等
- **8 个绝对禁止**：不得保证盈利、不得诱导下注、不得鼓励高风险行为等

#### 语言与表述规范 / Language & Expression Standards
- 推荐用语列表 (✅)
- 禁用语言列表 (❌)
- 明确了如何谈论期望值、风险和不确定性

#### 风险保护机制 / Risk Protection Mechanisms
- **单笔风险上限**: 保守 ≤2%、中性 ≤5%、激进 ≤10%
- **总风险敞口上限**: 保守 ≤5%、中性 ≤15%、激进 ≤30%
- **强制警告触发条件**: 6 种场景自动触发风险警告

### 4. 沟通指南 (Communication Guidelines)

新增五大沟通原则：

1. **开场与定位**: 每次明确说明角色和性质
2. **主动询问确认**: 数据来源、用户预算、风险偏好、分析目标
3. **渐进式披露**: 根据用户需求调整输出详细程度
4. **纠正性反馈**: 温和但坚定地纠正不当表述
5. **不确定性强调**: 在关键结论处反复强调

### 5. 使用场景示例 (5 Example Scenarios)

为常见用户请求提供了详细的回应流程：

- Scenario 1: 基础概率查询
- Scenario 2: 市场价值扫描
- Scenario 3: 完整风险分析报告
- Scenario 4: 彩票参考列表
- Scenario 5: 锦标赛冠军概率

### 6. 质量检查清单 (Quality Checklist)

每次输出前必须检查的 5 个维度：

- 内容完整性 (5 项)
- 风险披露 (5 项)
- 语言合规 (5 项)
- 技术准确性 (5 项)
- 用户体验 (5 项)

## 关键设计原则 / Key Design Principles

### 1. 教育优先 / Education First
所有输出都是"教育性示例"和"研究参考"，不是"投资建议"或"下注指令"。

### 2. 透明度优先 / Transparency First
- 展示所有计算公式
- 说明所有假设和局限
- 标注所有数据来源和时间戳
- 公开审计追踪 (blind-commit)

### 3. 风险优先 / Risk First
- 默认推荐保守方案
- 同时展示最佳和最坏情况
- 强制风险上限和警告触发
- 强调方差和不确定性

### 4. 用户自主 / User Autonomy
- 不替用户做决策
- 提供多种方案供选择
- 纠正认知但尊重最终选择
- 明确决策责任在用户

### 5. 合规优先 / Compliance First
- 严格禁用诱导性语言
- 要求遵守当地法律法规
- 拒绝不当请求并说明原因
- 保护用户避免高风险行为

## 技术增强 / Technical Enhancements

### 概率计算透明化
- 明确展示去水公式: power method vs proportional
- 分离呈现: 市场概率、模型概率、融合概率
- 标注融合权重: 默认 0.7 市场 + 0.3 模型

### EV 与 Kelly 重新定位
- 从"参考指标"升级为"教育性公式演示"
- 明确说明假设条件和适用限制
- 建议使用 fractional Kelly (1/4 或 1/8)
- 设置硬性上限 (≤10% per position)

### 情景建模
- 保守/中性/激进三种完整方案
- 每个方案包含：配置逻辑、示例配置、最佳/最坏情况、预期收益、破产风险
- 明确标注为"教育性示例"

### 风险因素分类
- 系统性风险: 模型风险、数据风险、市场风险、方差风险
- 比赛特定风险: 伤停、战术、天气、裁判、激励
- 组合风险: 相关性、流动性、最大回撤

## 预期效果 / Expected Outcomes

### 对用户
1. **更清晰的认知**: 理解预测市场是概率游戏，不是稳赚工具
2. **更好的决策**: 基于完整信息和风险权衡，而非盲目跟随
3. **更强的保护**: 避免高风险行为和认知偏差
4. **更高的透明度**: 了解分析的每一步逻辑和假设

### 对 Skill 本身
1. **更明确的定位**: 研究助手，而非下注代理
2. **更强的合规性**: 符合负责任 AI 的伦理准则
3. **更好的防御**: 拒绝不当请求的明确机制
4. **更高的质量**: 结构化输出和检查清单保证一致性

### 对生态
1. **树立标准**: 为预测市场分析工具设定负责任的范例
2. **降低风险**: 减少用户因过度自信或误解而遭受损失
3. **提升信任**: 通过透明和诚实建立长期可信度
4. **促进教育**: 帮助用户学习概率思维和风险管理

## 兼容性 / Compatibility

- 所有原有的技术计算能力

## 免责声明 / Disclaimer

本优化旨在将 skill 重新定位为负责任的研究和教育工具。然而：

1. **不能完全消除风险**: 即使有所有保护机制，用户仍可能做出高风险决策
2. **不能保证合规性**: 不同地区有不同法律，用户需自行确认合规
3. **不能替代专业建议**: 对于重大财务决策，应寻求专业顾问
4. **持续改进**: 本框架需要根据实际使用反馈不断迭代

| 隐含概率 | Implied Probability | 从市场赔率反推的概率 |
